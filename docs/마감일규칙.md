# 마감일 추출 규칙 (MVP-2) — 구현 근거

> 설계서 4장("마감일 추출 — ★핵심 로직")을 실제 코드로 옮긴 문서입니다.
> 코드: `src/extractor/deadline_parser.py`

---

## 0. 정의 (설계서 4장)

> **마감일 = 부장이 '액션(신청·제출·회신)' 해야 하는 기한. 행사일이 아님.**

검증에서 확인된 함정:
- AI(3b)가 마감일을 7.29 → 7.15 로 **지어냄**.
- 날짜가 둘일 때(연수일 3.10 vs 신청기한 3.6) **핵심을 못 집음**.
→ 그래서 날짜는 **규칙(정규식)**으로 뽑고, 문맥의 **액션 단어**로 성격을 가립니다. AI 안 씀.

---

## 1. 처리 흐름

```
본문 텍스트
  → ① 날짜를 모두 찾는다 (점 형식 2026. 7. 29. / 한글 2026년 7월 29일)
  → ② 각 날짜의 성격을 가린다 (아래 규칙)
  → ③ 마감일만 모아 '가장 이른 것'을 대표 마감으로
  → 결과: all_dates / deadlines / primary / has_deadline
```

## 2. 라벨 (설계서: 라벨 구분 필수)

| 라벨 | 뜻 | 판단 근거 단어(문맥) |
|---|---|---|
| **신청기한** | 신청·접수 기한 | 신청·접수·등록·응모·공모·모집 |
| **제출기한** | 제출·회신 기한 | 제출·회신·보고·납부·송부 |
| **마감** | 일반 기한 | 마감·기한·까지·완료 |
| **행사일** | 행사·일시(마감 아님) | 일시·행사·실시·개최·기간 |
| **참고** | 그 외 날짜(마감 아님) | (액션어 없음) 또는 문서 처리일 |

- **마감일 = 신청기한 · 제출기한 · 마감** 셋. 행사일·참고는 마감 아님.
- 세부 라벨은 구체적 단어(신청/제출)를 일반어(까지/마감)보다 **우선**합니다.
  예: "…까지 회신" → '까지'가 더 가깝지만 '회신'을 보고 **제출기한**으로.

## 3. 날짜가 둘일 때 (★핵심 — 검증된 함정)

같은 공문에 **연수일(행사)**과 **신청기한(마감)**이 함께 있으면:
- 각 날짜의 **앞뒤 문맥(약 25자)**에서 가장 가까운 단어로 따로 판단.
- "일시: 2026. 3. 10." → 행사일 / "신청: 2026. 3. 6.까지" → 신청기한(대표 마감).
- 실제 공문(doc31)에서 **3.6 을 정확히 대표 마감으로** 뽑음 (AI 가 틀린 그 케이스).

## 4. 기간(범위) 처리 — 「A ~ B」

공문엔 "신청기간: 2026. 6. 26. ~ 7. 7." 같은 **범위**가 흔합니다.
- 끝 날짜에 **연도가 생략**돼도(7. 7.) 시작 연도를 물려받아 인식.
- 사이에 **요일 (금)·시각 09:00**이 껴 있어도 건너뜀.
- **신청/제출** 액션어가 앞에 있으면 → **범위의 끝(B)이 마감**, 시작은 참고(기간 시작).
- **연수·행사 기간**(액션어 없음)이면 → 둘 다 **행사일**(마감 아님).

## 5. 오탐 방지 — 공문서 '문서 처리 날짜' 제외 ★중요★

모든 공문 하단(푸터)에는 이런 라우팅 정보가 있습니다:
```
시행  중등교육과-12985 (2026. 6. 29.)
접수  무릉초등학교-3015 (2026. 6. 29.)
```
- 여기 '접수'는 액션어처럼 보이지만 **문서 접수일**(메타데이터)입니다.
- 규칙: 날짜 바로 앞에 **`기관명-숫자(`** 형태가 있으면 → 문서 처리·참조 날짜로 보고 **마감에서 제외**.
- 이 규칙이 없으면 **모든 공문이 6월 말 접수일을 신청기한으로 오인**합니다. (실제로 발생 → 수정함)

## 6. 잘못된 날짜 걸러내기
- 문서번호 등에서 나온 `2026. 13. 40.` 같은 값은 실제 달력 날짜인지 확인해 버립니다.

---

## 7. 실제 공문 69개 검증 결과 (요약)
- 마감일 있는 공문: **26 / 69** — 나머지는 참고·규정·배포성(설계서 예측과 일치).
- 규정 안내(국가공무원법 개정 등)·배부 안내는 **정확히 '마감없음'**.
- 연수 신청 공문들은 **신청기한을 행사일과 구분해** 정확히 추출.

## 8. 알려진 한계 (Codex 교차검증 대상 — 설계서 10장)
- '까지'가 앞 문장에 걸치면 오판 가능 (설계서 명시).
- 연말~연초 범위(12월 ~ 1월)는 끝 날짜 연도 추정이 틀릴 수 있음.
- 표 안에 날짜가 흩어지면 문맥 25자로 못 잡을 수 있음 → 이후 정교화.
- 최종 대표 마감은 **사람이 확인**하는 것을 전제로 합니다(설계 원칙).
