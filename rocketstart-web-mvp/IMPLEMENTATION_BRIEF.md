# 로켓스타트 웹 MVP 1단계 개발 브리프

## 제품 정의

로켓스타트는 미루기를 많이 하는 사람이 일을 시작한 뒤 3일 안에 보여줄 수 있는 90% 산출물까지 가도록 돕는 실행 관리 앱이다.

중요한 원칙은 두 가지다.

1. 90% 완료는 뽀모도로 사용량이 아니라 필수 산출물 체크리스트 달성률로 판단한다
2. 뽀모도로는 계획 정확도를 학습하기 위한 예측과 실제 사용량 데이터로 추적한다

## MVP 목표

웹 앱만으로 다음이 가능해야 한다.

- 프로젝트 생성
- 로켓 태스크 생성
- 90% 산출물 체크리스트 생성
- 산출물별 하위 과업 생성
- 하위 과업별 예상 뽀모도로 입력
- 3일 스프린트 자동 계산
- 오늘 착수, 오늘 90% 마감, 복구 필요 화면 표시
- 뽀모도로 타이머 실행
- 백색소음 재생
- 실제 뽀모도로 세션 기록
- 예상 대비 실제 뽀모도로 분석
- 미룸 이유와 성찰 메모 기록

## MVP에서 제외할 것

- 네이티브 iOS 앱
- Apple Calendar 연동
- Android 앱
- 유료 결제
- 팀 협업

단, 데이터 구조는 나중에 iOS, Android, Chrome 확장으로 확장 가능하게 만든다.

## 핵심 데이터 관계

Project 1개에는 RocketTask 여러 개가 있다.
RocketTask 1개에는 Deliverable 여러 개가 있다.
Deliverable 1개에는 Subtask 여러 개가 있다.
Subtask에는 FocusSession 여러 개가 연결된다.
ReflectionLog는 RocketTask나 FocusSession에 연결된다.

## 90% 산출물 완료 계산

산출물마다 weight를 둔다.
필수 산출물만 90% 완료 계산에 넣는다.

필수 산출물 달성률:

completed_required_weight / total_required_weight

90% 완료 조건:

필수 산출물 달성률이 0.9 이상이면 90% 완료 후보로 판단한다.
사용자는 마지막에 90% 완료 확정 버튼을 누를 수 있다.

## 뽀모도로 추적 계산

예상 뽀모도로:

하위 과업에 입력된 estimated_pomodoros 합계

실제 뽀모도로:

완료된 focus_sessions의 completed_pomodoros 합계

사용률:

actual_pomodoros / estimated_pomodoros

예측 오차:

actual_pomodoros - estimated_pomodoros

예측 정확도:

1 - abs(actual_pomodoros - estimated_pomodoros) / estimated_pomodoros

단, estimated_pomodoros가 0이면 null 처리한다.

판단 라벨:

- 정확함: 실제가 예상의 75% 이상 125% 이하
- 과소예측: 실제가 예상의 125% 초과
- 과대예측: 실제가 예상의 75% 미만
- 데이터 부족: 실제 세션 없음 또는 예상 없음

## 화면 우선순위

### 1. 오늘 화면

오늘 시작할 로켓, 오늘 90% 마감, 복구 필요, 진행 중 로켓을 보여준다.

### 2. 로켓 만들기 화면

프로젝트, 목표, 90% 산출물, 하위 과업, 예상 뽀모도로, 로켓 시작일, 3일 계산 방식을 입력한다.

### 3. 집중 타이머 화면

15분 최소 시작, 25분 기본 뽀모도로, 50분 깊은 작업, 사용자 지정 시간을 제공한다.
화이트 노이즈, 브라운 노이즈, 빗소리, 카페 소리, 무음을 제공한다.

### 4. 복구 모드 화면

미착수, 과업 과다, 자료 부족, 에너지 부족, 두려움, 완벽주의, 일정 충돌 중 이유를 고르게 한다.
다음 15분 최소 행동을 만들게 한다.

### 5. 리포트 화면

프로젝트별, 로켓별 예상 뽀모도로와 실제 뽀모도로, 예측 정확도, 산출물 달성률을 보여준다.

## 핵심 제품 문장

미루는 사람이 시간을 잘 못 지키는 이유는 의지가 약해서가 아니라, 시작 기준과 산출물 기준과 복구 기준이 없기 때문이다. 로켓스타트는 이 세 가지 기준을 앱 안에 고정한다.
