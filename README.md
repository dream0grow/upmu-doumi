# 교무부장 도우미

학교 공문을 **100% 로컬**(내 PC 안)에서 읽어 마감일 추출·우선순위 정리·문서 작성을 돕는 프로그램.
데이터가 PC 밖으로 절대 나가지 않는 것이 핵심 가치입니다.

> 상세 설계는 설계 문서를 따릅니다. 개발 순서·원칙은 아래를 참고하세요.

---

## 지금까지: MVP-1(텍스트) + MVP-2(마감일) + MVP-3(교무수첩 카드) ✅

공문 파일 **1개**를 넣으면
1. **파일명**에서 발신기관·문서번호·제목 등을 규칙(정규식)으로 정확히 뽑고
2. **본문**을 형식에 맞는 파서로 읽어 텍스트로 만들고
3. 본문에서 **마감일을 규칙으로 추출**해 성격을 가리고
   (신청기한 / 제출기한 / 마감 / 행사일 / 참고),
4. 이 모두를 **교무수첩 카드**로 정리합니다
   (공문 4성격 분류 + D-day + 업무유형).

지원 형식(설계서에서 검증 완료): **PDF · HWP(구형) · HWPX(신형) · ODT · XLSX · XLS(구형)**

> 이 단계까지는 **AI를 전혀 쓰지 않습니다.** 정확한 사실(발신기관·문서번호·마감일)은
> 규칙으로 뽑아야 하기 때문입니다. (검증에서 AI가 발신기관·마감일을 지어내는 환각 확인됨)
> **로컬 LLM(Ollama)은 MVP-4(요약·할일 제안)에서 등장**하며, 그때도 PC 안에서만 돕니다.

## 설치 (처음 한 번)

```bash
# 1) 파이썬 가상환경 만들기 (내 PC를 깨끗하게 유지)
python3 -m venv .venv
source .venv/bin/activate        # 윈도우: .venv\Scripts\activate

# 2) 필요한 라이브러리 설치
pip install -r requirements.txt
```

## 사용법

```bash
source .venv/bin/activate

# 공문 파일 하나에서 텍스트 뽑기 (사람이 읽는 형식)
python -m src.extract "공문파일.hwp"

# 결과를 JSON 으로 받기 (나중에 다른 기능이 이어받기 좋음)
python -m src.extract "공문파일.pdf" --json
```

## 테스트 (코드가 잘 도는지 확인)

```bash
source .venv/bin/activate
pip install -r requirements-dev.txt
python -m pytest
```

실제 공문에는 개인정보가 있어 저장소에 올리지 않습니다(보안 원칙).
대신 테스트가 각 형식의 **진짜 샘플 파일을 그때그때 만들어** 파서를 검증합니다.

## 폴더 구조

```
├─ src/
│  ├─ parsers/         형식별 파서 (pdf·hwp·hwpx·odt·xlsx)
│  ├─ extractor/       규칙 기반 추출 (파일명 파싱)
│  └─ extract.py       텍스트 추출 총괄 + 명령줄 실행 입구
├─ tests/              실제 샘플을 만들어 검증
├─ docs/               파싱규칙 등 세부 문서
├─ requirements.txt        제품에 필요한 라이브러리
└─ requirements-dev.txt    테스트에만 필요한 라이브러리
```

## AI 요약·할일 (MVP-4, 선택) — 로컬 Ollama

규칙으로 못 하는 **요약·할 일 제안**은 로컬 LLM(Ollama)이 돕습니다.
**Ollama 는 선생님 PC 안에서만 돌며, 공문이 PC 밖으로 나가지 않습니다.**

```bash
# 선생님 PC 준비 (딱 두 번)
#   1) https://ollama.com 에서 Ollama 설치
#   2) ollama pull qwen2.5:3b

python -m src.extract "공문파일.hwp" --ai        # 규칙 결과 + AI 제안
python verify_ai.py "공문파일.hwp"                # 실제 Ollama로 검증
```

- 각 항목에 **[근거: 공문 해당 부분]**을 붙여 환각을 즉시 발각합니다.
- AI 출력엔 항상 "확인 필요" 표시. **마감일·기관명은 AI가 아니라 규칙이 정답.**
- Ollama 가 없어도 크래시하지 않고, 규칙 결과(마감일·성격)는 그대로 나옵니다.
