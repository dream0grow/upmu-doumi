# 파싱 규칙 (5형식) — MVP-1 구현 근거

> 설계서 3장("파일 파싱 — 검증 완료")을 실제 코드로 옮긴 세부 문서입니다.
> 각 형식이 **어떻게** 텍스트가 되는지, 왜 그렇게 했는지를 적었습니다.

---

## 0. 큰 그림

```
파일 1개
  ├─ (규칙) 파일명 파싱      → src/extractor/filename_parser.py
  └─ (형식별) 본문 파싱       → src/parsers/*.py
        └─ 확장자로 파서 선택 → src/extract.py 의 _PARSERS 표
```

- **AI 없음.** 이 단계는 100% 규칙/라이브러리 기반입니다.
- 결과는 `ExtractResult`(파일명 정보 + 본문 텍스트)로 묶여 나옵니다.

## 1. 파일명 파싱 (`extractor/filename_parser.py`)

실제 공문 파일명 구조:

```
(무릉초등학교-2974 (첨부) 제주특별자치도교육청 체육건강과) 2026년 전국초등교원체육연수 계획.hwp
 └── 수신처 ──┘└문서번호┘└본문/첨부┘└──── 발신기관 ────┘ └──────── 제목 ────────┘ └확장자┘
```

- 정규식 하나로 수신처·문서번호·본문/첨부·발신기관·제목·확장자를 뽑습니다.
- 규칙에 안 맞는 평범한 파일명이면 `matched=False`로 두고 제목·확장자만 채웁니다.
- **발신기관을 AI로 뽑지 않는 이유**: 검증에서 3b 모델이 발신기관을
  '교육지원센터'로 지어내는 환각이 있었음 → 파일명 규칙 추출이 정확.

## 2. 형식별 본문 파서 (`parsers/`)

| 형식 | 파일 | 방법 | 핵심 포인트 |
|---|---|---|---|
| PDF(텍스트) | `pdf_parser.py` | pdfplumber | 글자가 거의 없으면 **이미지 PDF**로 보고 안내(OCR은 2차) |
| HWP(구형 5.0) | `hwp_parser.py` | olefile + zlib + 레코드파싱 | tag 67(PARA_TEXT), 제어문자·노이즈 필터 |
| HWPX(신형) | `hwpx_parser.py` | zipfile + section*.xml | `<hp:t>` 글자 추출 |
| ODT | `odt_parser.py` | zipfile + content.xml | 문단(`text:p`) 단위 태그 제거 |
| XLSX | `xlsx_parser.py` | openpyxl | 시트·행·칸을 탭/줄바꿈으로 |

### 2-1. PDF
- `page.extract_text()` 결과를 이어 붙입니다.
- 글자 수가 너무 적으면(`< 10`) `ImagePdfError` → 상위에서 "이미지 PDF" 안내로 전환.

### 2-2. HWP (가장 까다로움)
1. `olefile`로 OLE 꾸러미를 엽니다.
2. `FileHeader`의 속성 플래그(0번 비트)로 **압축 여부** 확인.
3. `BodyText/Section0, 1 ...`을 압축이면 `zlib.decompress(raw, -15)`로 풉니다.
4. 4바이트 머리표(tag/level/size)를 읽어 **레코드**를 훑습니다.
   - `size == 0xFFF`이면 다음 4바이트가 진짜 길이.
5. **tag 67(PARA_TEXT)**의 UTF-16 글자에서 조판 제어문자를 걸러냅니다.
   - 인라인/확장 제어문자 = 자기 포함 **8글자(16바이트)** 건너뜀.
   - 문자 제어문자 = **1글자** 건너뜀(줄바꿈 10·13만 살림).
6. **노이즈 필터**: 한글·숫자가 하나도 없는 라틴 쓰레기 줄 제거.

> ⚠️ HWP는 파일 전체를 코드로 만들기 어려워, 테스트는 레코드 해독 로직을
> 합성 바이트로 직접 검증합니다(`tests/test_hwp_records.py`).

### 2-3. HWPX / ODT
- 둘 다 실은 zip 꾸러미. 안의 XML에서 글자만 추립니다.
- HWPX: `Contents/section0.xml ...`의 `<hp:t>` (이름공간 버전 차이에 대비해
  태그 끝이 `t`인지로 느슨하게 판단).
- ODT: `content.xml`의 `text:p`/`text:h` 문단, `tab`·`line-break`도 반영.

### 2-4. XLSX
- `data_only=True`로 **수식이 아닌 계산된 값**을 읽습니다.
- 시트 이름을 머리로, 행은 줄바꿈, 칸은 탭으로 이어 붙입니다.
- 완전히 빈 행은 건너뜁니다.

## 3. 알려진 한계 (다음 단계 과제)

- 이미지 PDF(포스터 등) → OCR 필요(2차 기능). 지금은 안내만.
- HWP 표/그림 안의 텍스트는 제어문자로 건너뛰므로 완벽하진 않음.
- 확장자 `.xls`(구형 이진 엑셀)는 openpyxl로 못 열 수 있음 → 필요 시 별도 처리.

## 4. Codex 교차검증 대상 (설계서 10장)
- HWP 파싱 노이즈 처리의 정확도
- 파일명 파싱이 예외적 이름에서 오판하지 않는지
