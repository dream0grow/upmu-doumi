# 📱 「100시간 공부」를 진짜 아이폰 앱으로 만들기 — 맥북 가이드

이 폴더는 `study/`에 있는 웹앱을 **진짜 아이폰 앱으로 포장**하는 프로젝트입니다.
아래 순서대로 따라 하면, 내 아이폰 홈 화면에 앱 아이콘이 생기고
**화면이 잠겨 있어도 타이머가 끝나는 순간 알림이 울립니다.**

> 처음 보는 단어가 나와도 겁먹지 마세요. 각 단계마다 "왜 하는지"를 함께 적었습니다.
> 순서대로 한 줄씩 하면 됩니다. 막히면 맨 아래 [문제 해결](#-문제-해결)을 보세요.

---

## 🧩 등장인물 소개 (이것만 알면 절반은 끝)

| 이름 | 하는 일 | 비유 |
|------|---------|------|
| **study/** 폴더 | 우리가 만든 웹앱 원본 (HTML/JS) | 요리(내용물) |
| **Capacitor** | 웹앱을 앱 모양으로 포장 + 아이폰 기능과 연결 | 도시락 통 + 통역사 |
| **Node.js / npm** | 포장 도구를 내려받고 실행하는 프로그램 | 배달부 |
| **Xcode** | 앱을 조립해서 아이폰에 설치하는 애플 공식 공장 | 공장 |
| **CocoaPods** | Xcode가 쓰는 부품 배달부 (npm의 애플 버전) | 공장 전용 배달부 |
| **서명(Signing)** | "이 앱은 내가 만들었어요" 도장. 애플이 요구함 | 인감 도장 |

**전체 흐름 한 장 요약:**

```
study/ (웹앱 원본)
   │  npm run copy        ← 원본을 www/ 상자에 복사
   ▼
ios-app/www/ (복사본)
   │  npx cap sync ios    ← 상자를 아이폰 앱 프로젝트에 넣기
   ▼
ios-app/ios/ (Xcode 프로젝트)
   │  Xcode에서 ▶ Run    ← 조립해서 아이폰에 설치
   ▼
📱 내 아이폰의 진짜 앱!
```

## 0단계. 준비물 확인

- 맥북 (완료! ✅)
- 아이폰 + 연결 케이블
- Apple ID (아이폰 쓰고 계시면 이미 있음)
- 시간 여유: 처음 설치는 1~2시간 (대부분 다운로드 기다리는 시간)

---

## 1단계. Xcode 설치 — 앱 공장 짓기

1. 맥북에서 **App Store** 열기 → "**Xcode**" 검색 → 받기
2. 용량이 크니(약 15GB) 기다리는 동안 2단계를 먼저 해도 됩니다.
3. 설치가 끝나면 Xcode를 **한 번 실행**하세요. "추가 구성요소 설치" 팝업이 나오면 설치를 눌러줍니다.

> **왜?** 아이폰 앱은 오직 Xcode로만 아이폰에 설치할 수 있습니다. 애플의 규칙이에요.

---

## 2단계. Node.js 설치 — 배달부 고용하기

1. https://nodejs.org 접속 → **LTS** 버전(왼쪽 큰 버튼) 다운로드 → 설치
2. 설치 확인: **터미널**을 열고(Spotlight에서 ⌘+스페이스 → "터미널" 검색) 아래를 입력

```bash
node --version
```

`v20` 이상 숫자가 나오면 성공입니다.

> **왜?** Capacitor(포장 도구)는 Node.js 세계의 도구라서, 그걸 내려받아 줄
> 배달부(npm)가 먼저 필요합니다. npm은 Node.js를 설치하면 같이 딸려 옵니다.
>
> **터미널이 뭐예요?** 마우스 클릭 대신 글자로 컴퓨터에게 명령하는 창입니다.
> 개발자가 가장 많이 쓰는 도구예요. 아래 명령들을 한 줄씩 복사해 붙여넣으면 됩니다.

---

## 3단계. 저장소 내려받기

터미널에서:

```bash
cd ~/Documents                                        # 문서 폴더로 이동
git clone https://github.com/dream0grow/rocketstart.git   # 저장소 통째로 복사
cd rocketstart/ios-app                                # 이 폴더로 이동
```

> **왜?** 지금 보고 있는 이 코드를 맥북으로 가져오는 과정입니다.
> `cd`는 "폴더 이동(change directory)", `git clone`은 "저장소 복제"라는 뜻이에요.
> 이미 클론해 두었다면 `cd` 이동 후 `git pull`로 최신 코드만 받으면 됩니다.

---

## 4단계. 부품 설치 — 배달부에게 주문서 넘기기

```bash
npm install
```

> **왜?** `package.json`이라는 주문서에 "Capacitor 본체, iOS 포장지, 로컬 알림 부품"이
> 적혀 있고, npm이 그걸 읽고 `node_modules` 창고에 전부 내려받습니다.
> 1~2분 걸리고, `added ... packages`가 나오면 성공입니다.

---

## 5단계. CocoaPods 설치 — 공장 전용 배달부

```bash
brew install cocoapods
```

`brew: command not found`라고 나오면 Homebrew(맥 프로그램 설치 도우미)부터 설치하세요:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

설치 후 터미널이 안내하는 마무리 명령 2줄(`echo ... >> ~/.zprofile`, `eval ...`)을
복사해 실행하고, 다시 `brew install cocoapods`를 실행합니다.

> **왜?** npm이 자바스크립트 부품 배달부라면, CocoaPods는 Xcode(애플 공장)용
> 부품 배달부입니다. Capacitor가 아이폰 프로젝트를 만들 때 이 배달부를 부릅니다.

---

## 6단계. 아이폰 프로젝트 생성 — 도시락 통 만들기

```bash
npm run copy        # study/ 원본을 www/ 상자에 복사
npx cap add ios     # www/ 를 담을 아이폰 프로젝트(ios 폴더) 생성
```

> **왜 copy를 먼저?** Capacitor는 `www/` 폴더를 앱에 담는데, 우리 원본은 `study/`에
> 있습니다. 그래서 복사부터 합니다. (원본을 두 군데 두면 헷갈리니 복사본은 저장소에
> 올리지 않도록 설정해 두었어요 — `.gitignore` 파일 참고)
>
> `npx cap add ios`는 `ios/`라는 폴더를 새로 만듭니다. 이 안에 진짜 아이폰 앱
> 프로젝트(Xcode용)가 들어 있습니다. **딱 한 번만** 실행하면 됩니다.

---

## 7단계. 웹 코드를 앱에 넣고 Xcode 열기

```bash
npm run sync        # 복사 + 앱 프로젝트에 반영 (앞으로 코드 바꿀 때마다 이것만!)
npm run open        # Xcode로 아이폰 프로젝트 열기
```

---

## 8단계. 서명 설정 — 도장 찍기 (Xcode에서, 처음 한 번만)

1. Xcode 왼쪽 파일 목록 맨 위의 파란 **App** 아이콘 클릭
2. 가운데 화면에서 **Signing & Capabilities** 탭 클릭
3. **Team** 옆 드롭다운 → **Add an Account...** → 본인 Apple ID로 로그인
4. Team을 방금 추가한 계정(Personal Team)으로 선택
5. 빨간 오류가 남아 있으면 **Bundle Identifier**를 세상에 하나뿐인 이름으로 살짝 바꿔주세요.
   예: `com.dream0grow.study100` → `com.leehg.study100`

> **왜?** 애플은 "누가 만들었는지 모르는 앱"이 아이폰에 설치되는 걸 막습니다.
> Apple ID로 도장(서명)을 찍으면 "내가 만든 앱을 내 폰에 설치"할 수 있게 허락해 줍니다.
> **무료**입니다. 다만 무료 도장은 7일짜리라, 일주일 뒤 앱이 안 열리면
> Xcode에서 ▶를 한 번 더 눌러 다시 설치하면 됩니다. (연 $99 유료 계정은 1년 유지 +
> 앱스토어 출시 가능 — 4단계 공부 때 결정하면 충분해요.)

---

## 9단계. 아이폰에 설치! 🚀

1. 아이폰을 케이블로 맥북에 연결 (아이폰에서 "이 컴퓨터를 신뢰" → 신뢰)
2. 아이폰에서 **개발자 모드** 켜기: 설정 → 개인정보 보호 및 보안 → 맨 아래 **개발자 모드** → 켬 → 재시동
   (이 메뉴는 Xcode에 한 번 연결된 뒤에야 나타납니다)
3. Xcode 상단 가운데 기기 선택에서 **내 아이폰** 선택
4. **▶ (Run)** 버튼 클릭!
5. 처음엔 아이폰에서 "신뢰하지 않는 개발자" 오류가 날 수 있어요:
   아이폰 설정 → 일반 → VPN 및 기기 관리 → 내 Apple ID → **신뢰**를 누르고 다시 실행

**성공하면**: 홈 화면에 「100시간 공부」 아이콘이 생깁니다.
타이머를 시작하면 알림 허용 팝업이 뜨고(허용!), 화면을 잠가도 끝나는 순간 알림이 울립니다. 🎉

```bash
# 1) study/ 폴더의 웹 코드를 수정한다 (맥북 아무 편집기나 OK)
# 2) 터미널에서:
cd ~/Documents/rocketstart/ios-app

# 3) Xcode에서 ▶ Run
```

웹 버전(GitHub Pages)과 아이폰 앱이 **같은 study/ 코드**를 쓰기 때문에,
한 번 고치면 양쪽 모두에 반영됩니다. 이게 Capacitor 방식의 가장 큰 장점이에요.

## 🔔 알림은 어떻게 동작하나요? (코드 공부 포인트)

- `study/js/notify.js` — 새로 만든 부품. 타이머 시작 때 iOS에게
  "○시 ○분에 알림 울려줘"라고 **예약**하고, 멈추면 **취소**합니다.
  화면이 잠겨 있어도 울리는 이유: 알림은 우리 앱이 아니라 **iOS가 대신** 울려주기 때문입니다.
- `study/js/timer.js` — 시작(`resume`)·일시정지(`pause`)·정지(`stop`)에서
  notify를 부르는 세 줄이 추가되었습니다. 직접 찾아보세요!
- 웹 브라우저에서는 notify.js가 "빈 껍데기"로 동작해서 아무 영향이 없습니다.
  이렇게 "환경에 따라 조용히 꺼지는 코드"를 만드는 게 여러 환경을 지원하는 요령입니다.
- 앱을 **열어둔 채** 타이머가 끝나면 완료음(beep.js)과 알림이 거의 동시에 울릴 수 있어요.
  거슬리면 알림 문구만 보이게 다듬을 수 있습니다 — 다음 공부거리로 남겨둡니다.

## 🚑 문제 해결

| 증상 | 처방 |
|------|------|
| `command not found: node` | 2단계 Node.js 설치를 다시. 설치 후 터미널을 **새로** 열기 |
| `command not found: brew` | 5단계의 Homebrew 설치 명령 실행 |
| `pod: command not found` 또는 `cap add ios`에서 pod 오류 | `brew install cocoapods` 후 다시 시도 |
| Xcode에서 Team 선택해도 빨간 오류 | Bundle Identifier를 나만의 이름으로 변경 (8단계 5번) |
| 아이폰이 Xcode 기기 목록에 안 보임 | 케이블 재연결, 아이폰 잠금 해제, "이 컴퓨터를 신뢰" 확인 |
| "신뢰하지 않는 개발자" | 아이폰 설정 → 일반 → VPN 및 기기 관리 → 신뢰 |
| 일주일 뒤 앱이 안 열림 | 무료 서명 만료. Xcode에서 ▶ 한 번 더 (8단계 참고) |
| 앱은 뜨는데 화면이 하얗다 | `npm run sync`를 빼먹었을 가능성. 실행 후 다시 Run |
| 빌드 실패: `Sandbox: bash(...) deny(1) file-read-data ... Pods-App-frameworks` | Xcode가 빌드 스크립트를 모래상자에 가둬서 생기는 문제. TARGETS → App → **Build Settings** 탭 → 검색창에 `sandbox` → **User Script Sandboxing**을 **No**로 변경 후 다시 Run. (그래도 나면 PROJECT 쪽 App에서도 동일하게 No) |
| ⚠️ 노란 경고 여러 개 (`WKProcessPool is deprecated` 등) | 무시해도 됩니다. 플러그인 내부의 낡은 표현 알림일 뿐 동작에는 문제 없음 |
