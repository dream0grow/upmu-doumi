# AI 요약·할일 규칙 (MVP-4)

> 설계서 5장("AI 프롬프트 — 환각 억제")을 코드로 옮긴 것입니다.
> 코드: `src/ai/`

---

## 0. 역할 분담 (환각 원천 차단 — 검증으로 확정)

| 종류 | 담당 | 이유 |
|---|---|---|
| 마감일·발신기관·문서번호 | **규칙 추출** (MVP-1·2) | AI가 지어냄(환각). 규칙이 정확 |
| 한 줄 요약·해야 할 일 | **AI 제안** (MVP-4) | 창의적 정리는 AI가 잘함 |

→ AI는 **참고 제안**만. 마감일·기관은 절대 AI 값을 쓰지 않습니다.

## 1. 로컬 원칙

- Ollama 는 **선생님 PC 안에서(localhost:11434)** 도는 로컬 LLM.
- 클라우드 API 가 아니라 **공문이 PC 밖으로 나가지 않습니다**.
- 파이썬 표준 `urllib` 로 REST 호출만 함 (추가 라이브러리 없음, localhost 라 프록시 안 탐).
- 기본 모델 **qwen2.5:3b** (설계서: 교무실 표준 PC에서 속도 검증 통과). `--model` 로 변경 가능.

## 2. 환각 억제 프롬프트 (설계서 5장 원문)

```
너는 학교 공문을 정리하는 보조도구다. 아래 규칙을 반드시 지켜라.
1. 공문에 없는 내용은 절대 추가하지 마라. 있는 내용만 사용하라.
2. 날짜·기관명·숫자는 공문에 적힌 그대로만 쓰고, 추측하지 마라.
3. 각 요약·할일 항목 끝에 [근거: 공문에서 해당 부분]을 표시하라.
4. 확실하지 않으면 "공문에 명시 안 됨"이라고 쓰고 지어내지 마라.
```

- **근거표시가 핵심**: 3b가 "학업성적 시스템 개선"처럼 부풀린 적이 있는데,
  각 항목에 `[근거: ...]`를 붙이면 지어낸 것이 즉시 발각됩니다.
- 사실 왜곡을 줄이려 `temperature=0.2`(낮은 창의성)로 호출.
- 본문이 너무 길면 앞 4000자만 사용(3b 속도·안정성).

## 3. 출력엔 항상 '확인 필요' 표시

모든 AI 결과에 다음 안내가 붙습니다:
> ⚠ AI가 제안한 내용입니다. 반드시 확인해 주세요. (마감일·기관명은 규칙 추출이 정답)

## 4. Ollama 가 없을 때

- 크래시하지 않고, "Ollama 실행 시 제공됩니다" 안내만 하고 넘어갑니다.
- 마감일·성격 등 **규칙 결과(MVP-1~3)는 AI 없이도 그대로** 나옵니다.

## 5. 사용법

```bash
# 규칙 결과 + AI 제안까지 (Ollama 실행 필요)
python -m src.extract "공문파일.hwp" --ai
python -m src.extract "공문파일.hwp" --ai --model qwen2.5:7b

# 선생님 PC에서 실제 Ollama로 검증
python verify_ai.py "공문파일.hwp"
```

### 선생님 PC 준비 (딱 두 번)
1. https://ollama.com 에서 Ollama 설치
2. `ollama pull qwen2.5:3b`

## 6. 테스트 (Ollama 없이도 검증)

LLM 호출부를 '가짜 함수'로 주입할 수 있어, Ollama 없이 프롬프트 생성·응답 파싱을
단위 테스트로 검증합니다 (`tests/test_ai.py`). 실제 모델 품질은 `verify_ai.py`로 확인.

## 7. 알려진 한계 (설계서)
- "요약·할일의 완전 무인화"는 어떤 모델로도 100% 불가. 근거표시+엄격 프롬프트로 최대한 근접.
- 3b는 이따금 과장 경향 → 근거표시로 검출, 사람이 최종 채택.
