# 뽀모도로 예측 정확도 추적 설계

## 핵심 원칙

90% 완료는 산출물 체크리스트로 판단한다.
뽀모도로는 계획을 얼마나 잘 예측했는지 확인하는 학습 데이터로 사용한다.

## 추적해야 하는 값

| 값 | 의미 |
|---|---|
| estimated_pomodoros | 과업을 시작하기 전에 예상한 집중 단위 |
| completed_pomodoros | 실제 완료한 집중 단위 |
| usage_ratio | 실제 / 예상 |
| error | 실제 - 예상 |
| accuracy_score | 1 - 절대오차 / 예상 |
| estimate_label | 정확함, 과소예측, 과대예측, 데이터 부족 |

## 예시

예상 4, 실제 7이면 과소예측이다.
이 일은 생각보다 오래 걸렸다.
다음에는 비슷한 과업의 예상치를 늘려야 한다.

예상 8, 실제 4이면 과대예측이다.
이 일은 생각보다 쉬웠다.
다음에는 더 작게 잡거나 한 로켓에 묶어도 된다.

예상 6, 실제 7이면 정확함이다.
계획 감각이 안정적이었다.

## 리포트 질문

- 내가 자주 과소예측하는 과업은 무엇인가
- 내가 자주 과대예측하는 과업은 무엇인가
- 미루는 과업은 실제로 오래 걸리는가, 아니면 시작 저항이 큰가
- 15분 최소 시작을 쓴 과업은 복구율이 높아졌는가
- 산출물 달성률은 높은데 뽀모도로가 많이 든 작업은 무엇인가
- 뽀모도로는 적게 썼는데 산출물 달성률이 낮은 작업은 무엇인가

## 제품 인사이트

뽀모도로 데이터는 사용자를 평가하기 위한 점수가 아니다.
사용자가 자신의 예측 습관을 배우게 하는 거울이다.
