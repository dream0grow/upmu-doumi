# 디자인 가이드 — 로켓스타트

> 스타터(`rocketstart-web-mvp/app/globals.css`)가 이미 흑백 담백 스타일(카드 20px radius, progress bar 등)로 되어 있습니다. 이 가이드는 그 스타일을 유지하면서 "로켓 발사" 톤 포인트를 어디에 얼마나 넣을지, 그리고 화면별 마이크로카피를 정리합니다. 새로 디자인 시스템을 만드는 문서가 아니라, **이미 있는 CSS 변수 위에 규칙을 얹는 문서**입니다.

## 1. 색

### 이미 정의된 기본 토큰 (건드리지 않음)

```css
:root {
  color-scheme: light;
  --bg: #f7f7f8;      /* 배경 */
  --card: #ffffff;    /* 카드 배경 */
  --text: #18181b;    /* 기본 텍스트 */
  --muted: #71717a;   /* 보조 텍스트 */
  --line: #e4e4e7;    /* 테두리 */
}
```

### 새로 추가할 포인트 컬러 2개

로켓 발사 은유를 위한 최소한의 색만 추가합니다. 기존 흑백 톤을 깨지 않도록 **버튼 배경 전체를 바꾸지 않고, 강조가 필요한 아주 좁은 영역에만** 씁니다.

```css
:root {
  /* 기존 토큰 아래에 추가 */
  --rocket-orange: #ff5a1f;   /* 발사·진행·D-day 강조. "지금 움직여야 한다"는 신호 */
  --rocket-orange-bg: #fff1ea; /* 위 색의 옅은 배경(뱃지, 카드 강조 영역) */
  --recovery-blue: #4c6ef5;    /* 복구 모드 전용. 다그치지 않는 차분한 톤 */
  --recovery-blue-bg: #eef1ff; /* 복구 모드 배경 강조 */
}
```

**사용 규칙**
| 용도 | 색 | 비고 |
|---|---|---|
| 오늘 90% 마감 뱃지, D-day 숫자, 90% 완료 확정 버튼 | `--rocket-orange` | 기본 버튼(`--text` 검정)과 구분해서 "지금 액션이 필요함"을 표시 |
| 진행 중 progress bar 채움색 (기본은 검정 유지, 90%↑일 때만) | `--rocket-orange` | 90% 임박 시에만 오렌지로 전환, 그 전엔 기존 `--text` 검정 유지 |
| 복구 모드(`/recovery`) 전체 배경 톤, 버튼 | `--recovery-blue`, `--recovery-blue-bg` | 절대 오렌지·빨강 계열 쓰지 않기. 경고색은 "잘못했다"는 인상을 줌 |
| 그 외 모든 기본 UI | 기존 흑백 토큰 | 포인트 컬러는 화면 전체의 5% 이내로 제한 |

색을 절대 쓰지 않는 곳: 에러 메시지(기존 텍스트 색 유지 + ⚠ 아이콘으로 충분), 복구 모드의 "막힘 이유" 버튼 배경(선택 전에는 흑백, 선택 후에만 `--recovery-blue-bg`).

## 2. 타이포그래피

스타터가 시스템 폰트 스택을 그대로 씁니다. 새 폰트를 추가하지 않습니다.

```css
font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

| 요소 | 크기 | 굵기 | 비고 |
|---|---|---|---|
| 화면 제목 (`h1`) | 24px | 700 | 화면당 1개만 |
| 카드 제목 (`h2`, `h3`) | 18px | 600 | `.card h2, .card h3 { margin-top: 0; }` 이미 적용됨 |
| 본문/설명 (`.muted`) | 14px | 400 | `--muted` 색 |
| 뱃지(`.badge`) | 13px | 500 | 기존 `.badge` 클래스 그대로 |
| 숫자 강조 (D-day, 뽀모도로 카운트, 타이머) | 20~40px | 700 | 타이머 화면 숫자는 32px 이상 권장 |

## 3. 컴포넌트 (기존 클래스 재사용 우선)

이미 `globals.css`에 정의된 클래스: `.shell`, `.nav`, `.grid`, `.card`, `.row`, `.stack`, `.input`/`.textarea`/`.select`, `.button`/`.button.secondary`, `.badge`, `.progress`. **새 컴포넌트를 만들기 전에 이 클래스로 먼저 시도**하세요.

### 로켓 카드 (오늘 화면)
- 기본 골격: `.card.stack`
- 안에 스테이지 뱃지(`.badge`) + progress bar(`.progress`) + 예상/실제 뽀모도로 `.row` + 액션 버튼(`.button`)
- 90% 마감 버킷 카드만 상단 테두리를 `--rocket-orange` 2px로 강조 (`border-top: 3px solid var(--rocket-orange)`), 다른 버킷은 기본 `--line` 유지

### 진행바 (Deliverable 진행률)
- 기존 `.progress` 재사용, 채움 div 배경만 상황별로 교체
  - 0~89%: `background: var(--text)` (기존 검정 유지)
  - 90% 이상: `background: var(--rocket-orange)`

### 복구 모드 전용 카드
- `.card`에 `background: var(--recovery-blue-bg)`, `border-color: var(--recovery-blue)` 오버라이드한 `.card.recovery` 클래스 하나만 추가
- 막힘 이유 버튼: 기본은 `.button.secondary`, 선택된 것만 `background: var(--recovery-blue); color: white;`

### 타이머 원형 진행
- 기존 클래스에 없으므로 신규. `.timer-ring` 하나만 추가하고 내부 숫자는 `.card` 안에 배치해 재사용
- 진행 중에는 `--text`, 마지막 10% 남았을 때만 `--rocket-orange`로 전환 (급하다는 인상보다 "곧 끝난다"는 신호)

### 뱃지 색상 매핑 (스테이지)
| 스테이지 | 뱃지 스타일 |
|---|---|
| 예정 / 1일차 구조 잡기 / 2일차 핵심 제작 | 기존 `.badge` 그대로 (회색) |
| 3일차 90% 완성 | `.badge` + `border-color: var(--rocket-orange); color: var(--rocket-orange);` |
| 복구 필요 | `.badge` + `border-color: var(--recovery-blue); color: var(--recovery-blue);` |
| 최종 완료 | `.badge` + `background: var(--text); color: white;` |
| 보류 | 기존 `.badge` 그대로, 텍스트만 흐리게 |

## 4. 마이크로카피 규칙

**원칙: 평가하지 않는다. 로켓 은유를 과하게 쓰지 않는다(화면당 1~2곳).**

| 상황 | 쓰지 않을 표현 | 쓸 표현 |
|---|---|---|
| 로켓 생성 완료 | "저장되었습니다" | "로켓 발사 준비 완료" |
| 90% 산출물 달성 | "완료율 90% 이상 도달" | "90%까지 왔어요. 확정할까요?" |
| 뽀모도로 사용 초과(과소예측) | "예측 실패", "계획을 못 지켰습니다" | "생각보다 시간이 더 걸렸어요. 다음엔 이만큼 더 잡아볼까요?" |
| 복구 모드 진입 | "왜 안 했나요?", "밀렸습니다" | "멈춘 건 실패가 아니에요. 다시 불을 붙이는 질문 두 개만 할게요." |
| 복구 저장 완료 | "기록되었습니다" | "재점화 완료. 여기서부터 다시 시작해요." |
| 리포트 화면 도입부 | "당신의 성과 평가" | "예측 습관을 보는 거울이에요. 잘하고 못하고가 아니라 패턴을 봐요." |
| 빈 상태(로켓 없음) | "데이터가 없습니다" | "아직 발사 대기 중인 로켓이 없어요." |
| 에러 상태 | "오류가 발생했습니다" | "잠시 문제가 생겼어요. 다시 시도해 주세요." (기술적 원인 노출 금지) |
| 타이머 완료 | "타이머 종료" | "세션 완료! 25분 집중, 1뽀모도로로 기록했어요." |

**금지 표현**
- "실패", "밀렸다", "왜 안 했나", "게을렀다" 계열 — 절대 쓰지 않음
- 느낌표 남발 금지 (완료·발사처럼 긍정적 순간에만 1개)
- 복구 모드에서는 굵은 글씨·빨간 텍스트·경고 아이콘(⚠) 쓰지 않음. 정보성 아이콘(🔥, 🚀)만 허용

## 5. 여백·모서리·그림자 (기존 값 유지)

```css
border-radius: 20px;   /* .card */
border-radius: 12px;   /* input, textarea, select, button */
border-radius: 999px;  /* nav 버튼, badge, progress bar (완전한 pill) */
box-shadow: 0 8px 24px rgba(0,0,0,0.04); /* .card 그림자, 매우 옅게 유지 */
padding: 18px;  /* .card 내부 */
padding: 24px;  /* .shell, 640px 이하에서는 16px */
```

새 컴포넌트를 만들 때도 이 네 가지 수치(20px/12px/999px, 그림자 0.04 opacity)를 그대로 따르면 스타터와 이질감이 없습니다.

## 6. 접근성 최소 기준

- 색만으로 상태를 구분하지 않기 — 뱃지에는 항상 텍스트(스테이지명) 동반
- 버튼 터치 영역 최소 44px 높이 (모바일 기준, 기존 `.button` padding 12px + 폰트 크기면 충족)
- 복구 모드 버튼은 키보드 탭 이동 순서가 막힘 이유 나열 순서와 같아야 함
