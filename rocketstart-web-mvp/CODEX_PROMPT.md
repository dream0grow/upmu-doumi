# Codex용 첫 구현 프롬프트

이 저장소는 로켓스타트 웹 MVP 1단계 스타터다.
목표는 노션 없이 작동하는 미루기 복구형 일정관리 웹 앱을 만드는 것이다.

## 제품 원칙

1. 90% 완료는 필수 산출물 체크리스트 달성률로 판단한다
2. 뽀모도로는 완료 기준이 아니라 계획 예측 정확도 추적 데이터다
3. 미루면 미완료 목록에 방치하지 않고 복구 질문으로 전환한다
4. 웹 MVP를 먼저 만들고, 이후 iOS 앱과 Android 앱으로 확장한다

## 첫 작업

다음 순서로 구현해줘.

1. 프로젝트가 빌드되도록 누락된 파일과 타입 오류를 고친다
2. `lib/rocket/metrics.ts`의 함수에 단위 테스트를 추가한다
3. `/today` 화면에 mock rocket task 3개를 표시한다
4. `/new-rocket` 폼에서 산출물과 하위 과업, 예상 뽀모도로를 입력할 수 있게 한다
5. `/timer` 화면에서 15분, 25분, 50분 타이머와 백색소음을 동작하게 한다
6. `/report` 화면에서 예상 뽀모도로, 실제 뽀모도로, 사용률, 예측 정확도, 산출물 달성률을 보여준다

## 제약

- 아직 Supabase 실연동을 강제로 끝내지 않아도 된다
- mock 데이터 기반으로 먼저 UI와 계산 로직을 검증한다
- 새 라이브러리는 필요한 경우에만 추가한다
- secret 값은 코드에 넣지 않는다
- 노션 연동은 만들지 않는다
- Chrome 확장 프로그램은 아직 만들지 않는다

## 완료 기준

- `npm run typecheck` 통과
- `npm run lint` 통과
- `npm run test` 통과
- 주요 화면이 모바일 폭에서도 볼 수 있어야 한다
- 뽀모도로 예측 정확도 계산 결과가 리포트에 보인다
